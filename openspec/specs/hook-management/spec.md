## Purpose

Define how Pulse manages the Claude Code `SessionStart` hook in `.claude/settings.json` without breaking user-defined hooks.

## ADDED Requirements

### Requirement: Add SessionStart hook
The `addHook()` function in `hook-manager.ts` SHALL read `.claude/settings.json` (relative to `process.cwd()`), locate or create the `hooks.SessionStart` array, and append a hook entry with `"_pulse": true` and a `"command"` property generated by `platform.ts`'s `getHookCommand()`. The function SHALL NOT modify any existing hook entries in the array.

#### Scenario: Add hook to empty settings file
- **WHEN** `addHook()` is called and `.claude/settings.json` does not exist
- **THEN** the file is created with `{ "hooks": { "SessionStart": [{ "_pulse": true, "command": "<platform-command>" }] } }`

#### Scenario: Add hook to settings without hooks section
- **WHEN** `addHook()` is called and `.claude/settings.json` exists with `{ "permissions": {} }` but no `hooks` key
- **THEN** the `hooks.SessionStart` array is added containing the pulse entry, and the `permissions` key is preserved

#### Scenario: Add hook to settings with existing user hooks
- **WHEN** `addHook()` is called and `.claude/settings.json` has `{ "hooks": { "SessionStart": [{ "command": "user-script.sh" }] } }`
- **THEN** the pulse entry is appended to the `SessionStart` array and the existing user hook remains untouched

#### Scenario: Add hook to settings with existing hooks on other events
- **WHEN** `addHook()` is called and `.claude/settings.json` has `{ "hooks": { "PreToolUse": [{ "command": "lint.sh" }] } }`
- **THEN** a new `SessionStart` array is created with the pulse entry and the `PreToolUse` hooks are preserved

### Requirement: Idempotent hook addition
The `addHook()` function SHALL check if a hook entry with `"_pulse": true` already exists in the `SessionStart` array. If found, the function SHALL skip adding a duplicate and return without modification.

#### Scenario: Skip when pulse hook already exists
- **WHEN** `addHook()` is called and `SessionStart` already contains an entry with `"_pulse": true`
- **THEN** no changes are made to the file

### Requirement: Remove SessionStart hook
The `removeHook()` function SHALL read `.claude/settings.json`, filter out the entry with `"_pulse": true` from the `SessionStart` array, and write the file back. If the `SessionStart` array becomes empty after removal, the `SessionStart` key SHALL be removed from the `hooks` object. If the `hooks` object becomes empty, the `hooks` key SHALL be removed.

#### Scenario: Remove pulse hook preserving user hooks
- **WHEN** `removeHook()` is called and `SessionStart` contains `[{ "command": "user.sh" }, { "_pulse": true, "command": "npx pulse sync..." }]`
- **THEN** only the `_pulse: true` entry is removed, and `SessionStart` retains `[{ "command": "user.sh" }]`

#### Scenario: Remove pulse hook from single-entry array
- **WHEN** `removeHook()` is called and `SessionStart` contains only the `_pulse: true` entry
- **THEN** the `SessionStart` key is removed from the `hooks` object

#### Scenario: Remove hook when hooks object becomes empty
- **WHEN** `removeHook()` is called and `SessionStart` is the only key in `hooks` and contains only the pulse entry
- **THEN** the `hooks` key is removed entirely from settings.json

#### Scenario: Remove hook when no pulse hook exists
- **WHEN** `removeHook()` is called and no `_pulse: true` entry exists in `SessionStart`
- **THEN** no changes are made to the file

#### Scenario: Remove hook when settings file does not exist
- **WHEN** `removeHook()` is called and `.claude/settings.json` does not exist
- **THEN** the function returns without error

### Requirement: Settings file creation
When `.claude/settings.json` does not exist, `addHook()` SHALL create the `.claude/` directory (if needed) and write a new settings file containing only the hooks section. The file SHALL use 2-space indented JSON formatting.

#### Scenario: Create settings file and directory
- **WHEN** `addHook()` is called and neither `.claude/` directory nor `settings.json` exist
- **THEN** both are created, and the resulting JSON is formatted with 2-space indentation

### Requirement: Hook command from platform
The hook entry's `command` property SHALL be the string returned by `platform.ts`'s `getHookCommand()` function, which provides the correct `npx` invocation for the current OS (Windows native vs Unix/WSL).

#### Scenario: Windows platform hook command
- **WHEN** `addHook()` is called on Windows (non-WSL)
- **THEN** the command uses `npx.cmd` as provided by `getHookCommand()`

#### Scenario: Unix platform hook command
- **WHEN** `addHook()` is called on Unix or WSL
- **THEN** the command uses `npx` as provided by `getHookCommand()`
